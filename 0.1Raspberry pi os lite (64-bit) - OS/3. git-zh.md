


## 202501 - .git目录太大


原理：利用git clone 下拉自己的私有仓库.下载后会包含该项目的完整的配置信息，修改文件后可以立即直接用git push提交。

思路：
备份的文件

清空

用git clone 下拉自己的私有仓库

然后 用备份的文件覆盖它；

最后push提交到自己的私有仓库

step by step

利用git clone 下拉前，由于是私有仓库，需要token。用下面的命令查看
git remote -v 
or
git config -l

查看和备份好项目参数，所有的参数都在，不用重新建立配置参数
